What is Ralph?

Think of Ralph as a meticulous craftsperson for your software projects. You describe what you want to build, and Ralph works through it methodically — reading your requirements, making a plan, and building feature by feature.

Describe Your Vision

Write what you want in plain language. Ralph understands requirements documents (PRDs) and breaks them down into manageable stories.

Watch It Work

Ralph picks up one story at a time, implements it, tests it, and commits the code. Each step is logged so you always know what's happening.

Save Time

No more context switching. Start a build, step away for coffee, and come back to find your feature implemented and committed.

How Ralph Works

Three simple steps from idea to implementation

1

PRD

Write a Product Requirements Document describing what you want to build. No fancy format needed — just clear, plain language.

2

Plan

Ralph reads your PRD and breaks it into ordered user stories. Each story is a small, focused piece of work.

3

Build

Ralph executes stories one by one, making commits as it goes. You can watch progress in real-time or check back when it's done.

Quick Start

Get up and running in 5 minutes

1

Install Ralph CLI

git clone https://github.com/AskTinNguyen/ralph-cli.git
cd ralph-cli && npm i && npm link
2

Install to your project

cd your-project
ralph install
3

Generate a PRD and start building

ralph prd
ralph plan
ralph build 5
