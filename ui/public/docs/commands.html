PRD (Product Requirements)

Create and manage product requirements documents

ralph prd

Beginner

Generate a Product Requirements Document with support for interactive and headless modes

Syntax
Terminal 
ralph prd ["feature description"] [--headless]
Flags
--headless Run in non-interactive mode for server/UI/background execution (prevents TTY conflicts). Required when called by UI server, CI/CD, or other agents.
Execution Modes

Interactive Mode (default): Agent runs with full terminal output when a human runs the command in a terminal.
Headless Mode (--headless): Agent runs with piped stdin for server/UI/background execution. Use when Claude Code agent executes ralph prd to avoid nested interactions.

Examples
Terminal 
# Human running in terminal (interactive)
ralph prd "Build a user authentication system"

# UI server or background process (headless)
ralph prd "Add dashboard feature" --headless

# Agent calling ralph (headless)
ralph prd "Automated test suite" --headless
Output

Creates a new folder .ralph/PRD-N/prd.md where N auto-increments (PRD-1, PRD-2, etc.)

See also
ralph plan ralph stream new Agent Guide (AGENTS.md)

Planning

Create implementation plans from your PRD

ralph plan

Beginner

Generate an implementation plan from your PRD

Syntax
Terminal 
ralph plan [n] [--prd=N|<path>] [--plan <path>]
Flags
--prd <path> Path to custom PRD file (default: auto-detected)
--plan <path> Path to output plan file
Examples
Terminal 
# Plan the latest PRD (default)
ralph plan

# Plan a specific PRD by number
ralph plan --prd=1

# Plan with custom paths
ralph plan --prd docs/prd-api.md --plan .ralph/api-plan.md
Output

Creates .ralph/PRD-N/plan.md with tasks for each user story, including scope, acceptance criteria, and verification commands.

See also
ralph prd ralph build

Building

Execute your plan and build features

ralph build

Beginner

Run build iterations to implement your plan

Syntax
Terminal 
ralph build <iterations> [options]
Flags
<iterations> Number of stories to process (required)
--no-commit Dry run without committing changes
--prd <N> Specify which PRD to build (e.g., --prd=1)
--agent <name> Override agent (claude, codex, droid)
--plan <path> Path to custom plan file
--progress <path> Path to custom progress file
Examples
Terminal 
# Run 5 iterations
ralph build 5

# Dry run (no commits)
ralph build 1 --no-commit

# Build specific PRD
ralph build 5 --prd=1

# Use a different agent
ralph build 3 --agent=codex
See also
ralph plan ralph stream build

Error Handling

Look up error codes and remediation steps

ralph error

Beginner

Look up error codes and view remediation steps

Syntax
Terminal 
ralph error [RALPH-XXX] [options]
Flags
--list List all available error codes
--category <name> Filter errors by category (CONFIG, PRD, BUILD, GIT, AGENT, STREAM, INTERNAL)
--auto-issue Show only errors that auto-create GitHub issues
Examples
Terminal 
# Look up specific error
ralph error RALPH-401

# List all errors
ralph error --list

# Filter by category
ralph error --list --category=BUILD

# Show auto-issue errors
ralph error --list --auto-issue
Output

Displays error code, category, description, and remediation steps. Error codes follow pattern RALPH-XXX where range indicates category (001-099: CONFIG, 100-199: PRD, 200-299: BUILD, etc.)

See also
ralph build ralph stream build

ralph stream init

Intermediate

Create an isolated git worktree for a stream

Syntax
Terminal 
ralph stream init <N>
Examples
Terminal 
# Initialize worktree for PRD-1
ralph stream init 1

# Initialize worktree for PRD-2
ralph stream init 2
Output

Creates a git worktree at .ralph/worktrees/PRD-N/ on a new branch ralph/PRD-N.

See also
ralph stream build ralph stream cleanup

ralph stream build

Intermediate

Run build iterations in a stream's worktree

Syntax
Terminal 
ralph stream build <N> <iterations> [options]
Flags
<N> Stream/PRD number (required)
<iterations> Number of stories to process (required)
--no-worktree Build directly without using worktree
--force Force build even if lock exists
Examples
Terminal 
# Build PRD-1 with 5 iterations
ralph stream build 1 5

# Build without worktree
ralph stream build 1 5 --no-worktree

# Run two streams in parallel
ralph stream build 1 5 &
ralph stream build 2 5 &
wait
See also
ralph stream init ralph stream merge ralph build

ralph stream verify-status

Advanced

Auto-scan git history and fix stale status markers

Syntax
Terminal 
ralph stream verify-status
Output

Checks git history against file markers and auto-creates missing .completed or .merged markers when commits exist. Git commits are the ultimate source of truth for PRD status - checkboxes are only progress markers. Correctly identifies PRDs completed via direct-to-main workflow (no worktree).

Status Detection

Hierarchy: running (lock file) ‚Üí merged (branch merged to main) ‚Üí completed (commits on main) ‚Üí in_progress (progress.md exists) ‚Üí ready (plan.md exists)
Two workflows: merged = worktree workflow with PR, completed = direct-to-main workflow

See also
ralph stream status ralph stream mark-completed ralph stream unmark-completed

UI & Monitoring

Web dashboard and visualization tools

ralph ui

Beginner

Launch the Ralph web dashboard for monitoring and managing PRDs

Syntax
Terminal 
ralph ui
Output

Starts the web UI server on http://localhost:3000. The dashboard provides real-time monitoring of PRD status, progress tracking, activity logs, and interactive controls for managing builds.

Features

View all PRDs with live status updates, inspect detailed progress logs, monitor build activity in real-time, create new PRDs via wizard interface, and access comprehensive documentation.
